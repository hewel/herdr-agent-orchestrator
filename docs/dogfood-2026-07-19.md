## Failures fixed during the cycle

1. A `fresh` Task with `related_task_id` selected the related Task's ended Session
   instead of the new live Session. Session selection now ignores the related
   Session for effective `Fresh` policy; the regression exercises a complete ended
   Session replacement lifecycle.
2. Codex child-thread `turn/started` and `turn/completed` notifications overwrote
   the bound root thread's lifecycle and could fail its Task. Codex notifications
   carrying a different `threadId` are now ignored; the process-adapter regression
   proves only the bound root turn becomes terminal.
3. Repeated live control required error-prone hand-built JSON-RPC commands. The
   identity-bound `scripts/live-acceptance/control.sh` now centralizes status,
   graph, Inbox, event, Worker start, generic tool calls, evidence capture, and
   verified daemon handoff.
4. Evidence capture used obsolete dependency column names. The projection now
   records the persisted dependency revision and snapshot attachment columns.

## Release blockers

- Managed Worker presence needs a heartbeat or expiring lease. After pane loss,
  stale `online` state prevents safe replacement; PID files are not sufficient.
- A broker handoff currently causes pane-resident Hosts to exit instead of
  reconnecting safely.
- An OMP Supervisor Event can remain `accepted` without a visible processed turn;
  the manual same-pane nudge proved processing but not automatic injection.
  Unsettled injection must not be inferred successful or blindly replayed.
- The OMP review, Correction reuse, Approval, and downstream `Approved` unlock were
  not completed in this cycle.
- Dogfood orchestration changed files outside the active implementation Task's
  declared scope. Approval correctly refused to release its repository lease;
  the later user-authorized cancel-and-reconcile preflight released it without
  claiming Approval.
- Codex `0.144.5` deferred the Herdr MCP tools from the direct tool list. The Worker
  successfully used the identity-bound MCP executable fallback, but direct tool
  discovery remains a provider-integration limitation to resolve.
